# DAPLink Tool v1.0.0

集成 PC 端工具，整合串口终端、自定义协议收发、WiFi 桥接（直连 + 中继）、RTT 查看器。

## 版本历史

| 版本 | 修改内容 |
|------|----------|
| **v1.0.0** | 初始版本。串口/TCP终端、HEX收发、自定义协议编辑器（组包/解包/CRC）、协议列表管理、接收解析、实时曲线绘制(缩放/CSV)、WiFi桥接(直连+中继)、elaphureLink Proxy内置、RTT查看器 |

> 版本号定义在 `daplink_tool.py` → `APP_VERSION`，修改该值会自动更新窗口标题。

## 功能

| 标签页 | 功能 |
|--------|------|
| **终端** | 串口/TCP 终端，支持 USB/网络(3241/3242)发送目标切换、HEX 显示/发送、时间戳、周期发送 |
| **协议** | 自定义 HEX 协议编辑器，支持协议列表管理、单次/周期发送、接收解析、实时曲线绘制 |
| **桥接** | WiFi 调试桥接 — 中继模式(跨网络) 或 直连模式(同网络)，elaphureLink Proxy 内置 |
| **日志** | 应用运行日志 |

## 快速启动

### 方法 1: Python 运行
```bash
pip install -r requirements.txt
python daplink_tool.py
```

### 方法 2: 打包为 exe（无需 Python）
```bash
build.bat
```
生成 `dist/DAPLinkTool.exe`，可直接分发。

## 使用流程

### 终端调试（串口 + RTT）
1. 打开"终端"标签页
2. 选择 DAPLink 的 COM 端口，设置波特率
3. 点击"打开"
4. I/O 通道选择是统一的：选中哪个通道，发送和接收都走该通道（`USB串口` / `网络串口` / `网络RTT`）
5. 右侧"扩展命令"可添加、删除、保存并双击快速发送

### 自定义协议收发
1. 打开"协议"标签页
2. **编辑协议字段**：
   - 添加字段，支持 4 种类型：`常量`（帧头/命令）、`长度`、`数据`、`校验`
   - 数据字段支持：raw / uint8 / int8 / uint16 / int16 / uint32 / int32 / float，大小端可选
   - 校验支持：SUM8、SUM16、XOR8、CRC16/MODBUS、CRC16/CCITT、CRC32
   - 长度/校验的覆盖范围支持 `all`（所有字段）或指定索引范围如 `0-3`
3. **协议列表管理**：
   - 编辑完成后点击"保存到列表"，协议将出现在左侧列表中
   - 支持单次发送、周期发送、编辑、删除
4. **接收解析**：
   - 勾选"启用解析"，工具会自动根据列表中所有协议的帧头匹配并解析接收数据
   - 解析结果实时显示在下方表格（字段名、HEX值、解码值、时间戳）
5. **数据曲线**：
   - 在接收表格中勾选需要绘制的字段
   - 点击"绘制曲线"打开实时图表窗口
   - 支持鼠标滚轮缩放（Y轴区域=Y缩放，X轴/绘图区=X缩放）
   - 支持暂停、清空、调整刷新率、保存 CSV

### WiFi 中继桥接（跨网络）
1. 打开"桥接"标签页
2. 选择"中继桥接"
3. 输入中继服务器主机和中继端口，再输入 6 位配对码
4. 点击"启动桥接"
5. 在 Keil 中选择 elaphureLink，IP=127.0.0.1，端口=3240

### WiFi 直连（同网络）
1. 打开"桥接"标签页
2. 选择"直接连接"
3. 输入 ESP32 的 IP 地址和无线调试端口（默认 3240）
4. 点击"启动桥接"
5. 在 Keil 中选择 elaphureLink，IP=127.0.0.1，端口=3240

### RTT 查看
1. 先在"桥接"中连接设备
2. 在"终端"标签页查看带 `[NET 3242]` 前缀的数据

## 配置文件

桥接参数支持点击"保存设置"手动保存；关闭工具时也会自动保存到 `config.ini`，下次启动自动加载。

## elaphureLink 集成

桥接页新增 `elaphureLink Proxy（内置）`：
- **内置 Proxy**（推荐）：纯 Python 实现，无需安装 elaphureLink.Wpf.exe
  - 点击"启动 Proxy"或勾选"桥接启动时自动启动 Proxy"
  - 自动创建 Windows 共享内存和命名事件，Keil RDDI DLL 直接识别
- **外部 exe（备用）**：如需使用原版 elaphureLink.Wpf.exe，可配置路径

Keil 中选择 elaphureLink 调试器，无需额外操作。
